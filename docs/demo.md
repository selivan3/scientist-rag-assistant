# Демонстрация RAG-помощника по рисункам курсовой

В этой демонстрации использованы все рисунки из курсовой работы. Они показывают полный путь работы RAG-помощника учёного: от общей схемы поиска по источникам до интерфейса, структуры знаний, результата ответа и агентной организации.

## 1. Общая схема RAG

![Общая схема RAG](figures/рисунок-1-rag.png)

Рисунок показывает базовую идею: вопрос пользователя сначала связывается с поиском по базе знаний, затем найденный контекст передаётся языковой модели, а ответ строится с опорой на источники.

## 2. Конвейер обработки источников

![Конвейер обработки источников](figures/рисунок-1-конвейер.png)

Здесь показан путь документа: загрузка источника, разбиение на фрагменты, сохранение в индекс и дальнейший поиск релевантных частей по вопросу исследователя.

## 3. Интерфейс помощника

![Интерфейс помощника](figures/рисунок-2-интерфейс.png)

Интерфейс демонстрирует рабочую область помощника: выбор источников, вопрос, ответ и место для заметок исследователя.

## 4. Подготовка данных

![Подготовка данных](figures/рисунок-2-подготовка.png)

Рисунок относится к этапу подготовки корпуса: источники приводятся к виду, пригодному для поиска, анализа и дальнейшего использования в ответах.

## 5. База знаний

![База знаний](figures/рисунок-3-знания.png)

Здесь показано, что помощник работает не с абстрактной памятью, а с организованной базой знаний: источниками, фрагментами, метаданными и связями между материалами.

## 6. Результат ответа

![Результат ответа](figures/рисунок-3-результат.png)

Этот рисунок показывает итог работы помощника: ответ формируется по найденным фрагментам, а не придумывается без источников.

## 7. Демонстрация RAG-сценария

![Демонстрация RAG-сценария](figures/рисунок-4-rag-демо.png)

Сценарий демонстрирует работу помощника на учебном примере: вопрос, поиск релевантной информации, подготовка ответа и отображение результата.

## 8. Чёрно-белая схема конвейера

![Чёрно-белая схема конвейера](figures/рисунок-4-конвейер-чб.png)

Схема фиксирует тот же процесс в более строгом виде: источник проходит через обработку, индексирование, поиск и генерацию ответа.

## 9. Структуризация материалов

![Структуризация материалов](figures/рисунок-5-структуризация.png)

Рисунок показывает, как RAG-помощник помогает превращать набор документов в структурированные исследовательские заметки.

## 10. Агентная организация

![Агентная организация](figures/рисунок-6-агенты.png)

Финальная схема показывает, как вокруг RAG-помощника можно построить агентный контур: поиск, анализ, проверка, структурирование и подготовка научного ответа.
